# 讀書筆記本系統 PRD (Product Requirements Document)

## 1. 專案概述
- **背景與動機**：學生經常閱讀各類書籍，但不一定有系統性地記錄閱讀心得與評價的習慣。此系統旨在幫助學生建立個人的數位閱讀筆記本，方便日後回顧與反思。
- **目標用戶**：學生
- **核心價值主張**：提供一個輕量、簡單易用的系統，讓學生能快速記錄讀過的書名、撰寫心得、給予評分，並能輕鬆檢索過去的閱讀紀錄。

## 2. 功能需求
以下為系統主要功能與對應的使用者故事 (User Stories)：

1. **記錄書籍資訊 (Must Have)**
   - 作為學生，我希望能夠新增一本書的書名等基本資訊，以便建立我的閱讀清單。
2. **撰寫閱讀心得 (Must Have)**
   - 作為學生，我希望能夠為每一本書記錄個人的閱讀心得，以便日後回顧書中的精華與反思。
3. **書籍評分功能 (Must Have)**
   - 作為學生，我希望能夠為讀過的書給予評分（例如 1 到 5 星），以便快速了解我對這本書的喜愛程度。
4. **搜尋功能 (Must Have)**
   - 作為學生，我希望能夠透過關鍵字（如書名）搜尋已記錄的書籍，以便快速找到特定的筆記。
5. **編輯與刪除筆記 (Should Have)**
   - 作為學生，我希望能夠編輯或刪除已經建立的讀書筆記，以便在想法改變或記錄錯誤時進行修正。

## 3. 非功能需求
- **技術限制**：
  - 後端框架：Flask
  - 前端模板：Jinja2 
  - 資料庫：SQLite
- **效能與安全考量**：
  - 系統應具備基本的資料驗證機制（如書名不可為空）。
  - 使用 Flask 內建的方法防範常見安全漏洞（例如防止 SQL Injection 與 XSS）。
  - 回應速度應保持流暢，搜尋與資料載入需在合理時間內完成。

## 4. MVP 範圍 (Minimum Viable Product)
- **Must Have (必須有)**：
  - 新增讀書筆記（包含書名、心得、評分）。
  - 檢視所有閱讀過的書籍列表。
  - 透過關鍵字搜尋特定書本記錄。
- **Should Have (應該有)**：
  - 編輯與刪除已建立的讀書筆記。
  - 依照評分高低或加入時間排序閱讀清單。
- **Nice to Have (可以有)**：
  - 加入書籍作者欄位、分類標籤等進階資訊。
  - 讀書進度追蹤（例如：閱讀中、已讀完）。
